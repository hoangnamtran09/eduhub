# Kế hoạch: Thiết kế lại dashboard admin theo kiểu Notion

## Mục tiêu
Thiết kế lại `src/app/(dashboard)/page.tsx` cho vai trò `ADMIN` theo phong cách Notion: bố cục nhiều block, dễ quét nhanh, mật độ thông tin cao hơn, ưu tiên dữ liệu thật của hệ thống thay vì chỉ đổi giao diện.

## Hiện trạng đã khảo sát
- Trang dashboard hiện tại dùng chung cho `STUDENT`, `PARENT`, `ADMIN` trong `src/app/(dashboard)/page.tsx`.
- Với `ADMIN`, trang mới chỉ hiển thị một block danh sách học sinh từ `GET /api/reports/study-time`.
- `GET /api/reports/study-time` cho admin hiện chỉ trả:
  - `students[]`
  - `totalStudySeconds`
  - `totalSessions`
  - `lastActive`
  - `gradeLevel`
- `GET /api/admin/students` có thêm dữ liệu hữu ích hơn:
  - `profile`
  - `studySessions`
  - `enrollments`
- Sidebar admin đã có cấu trúc điều hướng riêng và nhóm cha/con, không cần đổi thêm cho task này.

## Hướng thiết kế
Tách nhánh hiển thị `ADMIN` trong dashboard thành một workspace kiểu Notion với các khối sau:

1. Header quản trị
- Tiêu đề rõ ràng kiểu “Bảng điều khiển trung tâm”.
- 1-2 CTA phù hợp admin:
  - đi tới `Quản lý học sinh`
  - đi tới `Giao bài tập`

2. Quick stats dạng database properties
- Tổng số học sinh.
- Tổng thời lượng học đã ghi nhận.
- Tổng số phiên học.
- Số học sinh có hoạt động gần đây.
- Có thể thêm phân bố theo khối lớp nếu dữ liệu đủ gọn.

3. Main column kiểu Notion blocks
- Block `Tổng quan theo học sinh`:
  - danh sách/top học sinh học nhiều nhất
  - cột: tên, lớp, giờ học, số phiên, hoạt động gần nhất
- Block `Cần chú ý`:
  - học sinh chưa có hoạt động gần đây
  - học sinh chưa phân lớp
  - học sinh có thời lượng học thấp hoặc chưa có phiên học
- Block `Phân bố khối lớp`:
  - tóm tắt số học sinh theo `gradeLevel`
  - hiển thị dạng chip/list ngắn thay vì biểu đồ nặng

4. Sidebar kiểu context panel
- Block `Hoạt động hệ thống`:
  - số học sinh active trong 7 ngày gần nhất
  - số học sinh inactive
- Block `Tác vụ nhanh`:
  - link sang `admin/students`
  - link sang `admin/assignments`
  - link sang `admin/subjects`
- Block `Ghi chú quản trị` hoặc `Lưu ý hôm nay`
  - chỉ là UI tĩnh nhẹ nếu chưa có backend notes

## Dữ liệu cần bổ sung
Hiện `reports/study-time` chưa đủ để làm dashboard admin dày thông tin một cách sạch sẽ. Nên mở rộng API này thay vì gọi quá nhiều endpoint ở client.

### Đề xuất mở rộng `GET /api/reports/study-time` cho `ADMIN`
Trả thêm một object `summary` và các list đã xử lý sẵn:

- `summary.totalStudents`
- `summary.totalStudySeconds`
- `summary.totalSessions`
- `summary.activeStudents7d`
- `summary.inactiveStudents7d`
- `summary.unassignedGradeCount`

- `students[]` giữ nguyên nhưng có thể thêm:
  - `isActive7d`
  - `enrollmentCount`

- `gradeDistribution[]`
  - `{ gradeLevel, count }`

- `attentionStudents[]`
  - học sinh chưa có phiên học hoặc inactive lâu

- `topStudents[]`
  - top theo `totalStudySeconds`

Lý do:
- giữ logic tổng hợp ở server
- giảm xử lý rối trong client dashboard
- tránh phải fetch đồng thời nhiều API admin khác nhau chỉ để render dashboard

## Kế hoạch triển khai chi tiết

### Bước 1. Mở rộng dữ liệu cho admin dashboard
Sửa `src/app/api/reports/study-time/route.ts` để khi `authUser.role === "ADMIN"`:
- tính summary tổng hợp
- tính active/inactive theo `profile.lastActive`
- tạo phân bố khối lớp
- tạo danh sách top students
- tạo danh sách attention students

### Bước 2. Tách nhánh render admin trong dashboard
Trong `src/app/(dashboard)/page.tsx`:
- giữ luồng `STUDENT` và `PARENT` hiện tại
- tách riêng nhánh `ADMIN` thành các component nội bộ tối giản như:
  - `AdminDashboard`
  - `AdminQuickStats`
  - `AdminTopStudents`
  - `AdminAttentionList`
  - `AdminGradeDistribution`
  - `AdminActionPanel`

Không cần tách file mới nếu code còn kiểm soát được. Ưu tiên sửa tối thiểu trong 1 file trước.

### Bước 3. Thiết kế layout kiểu Notion
Bố cục đề xuất:
- hàng đầu: hero + CTA + meta badges
- tiếp theo: 4 stat cards
- content grid 3 cột:
  - trái rộng: top students + attention list
  - phải hẹp: grade distribution + quick actions + system snapshot

Nguyên tắc UI:
- nhiều block nhỏ, khoảng cách đều
- border nhẹ, nền trắng hơi trong suốt như workspace
- typography rõ cấp bậc, không lòe loẹt
- giữ nhất quán với dashboard hiện tại đã được chỉnh theo hướng Notion

### Bước 4. Bổ sung thông tin cần thiết cho admin
Ưu tiên các thông tin hữu ích thực sự cho vận hành:
- ai học nhiều nhất
- ai chưa hoạt động
- số học sinh chưa phân lớp
- khối lớp nào đông nhất
- tổng số phiên học của hệ thống
- các đường tắt thao tác quản trị

Không thêm dữ liệu giả hoặc block trang trí không có giá trị vận hành.

### Bước 5. Kiểm tra hiển thị responsive
Đảm bảo:
- mobile: stack theo chiều dọc
- desktop: 2-3 cột hợp lý
- list dài không làm vỡ layout

- Cần tránh làm hỏng dashboard của `STUDENT` và `PARENT` khi chèn nhánh admin mới.
